HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements that are nested inside other elements are called child elements and the element that holds all the children is known as the parent element.
  • In Semantic HTML elements, the name of the elements describes it's content and the role it plays on a webpage. Not all elements are semantic though.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • An element refers to the entire element from opening tag to closing tag.
  • A tag refers only to what appears insdie the angle brackets.
  • HTML "hypertext markup language": gives the content structure.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: commits all files in the current working branch and adds modifications to the staging area.
  • git commit -m "commit message": commits the files.
  • Always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work.
  • git push: pushes the changes we have locally to our remote GitHub branch.
  • A branch is a copy or version of the code that can be worked on independently in isolation so development changes do not disrupt other team members who are also working on the code and testing their solutions.
