HTML

  • HTML "hypertext markup language": gives the content structure.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements that are nested inside other elements are called child elements and the element that holds all the children, is known as the parent element.
  • In Semantic HTML elements, the name of the elements describes it's content and the role it plays on a webpage. Not all elements are semantic though.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • An element refers to the entire element from opening tag to closing tag.
  • A tag refers only to what appears inside the angle brackets.

CSS

  • CSS defines the colors, aethetics, visual layout and organization.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • 3 ways to style a webpage using CSS: Inline CSS, Internal CSS style sheet and External CSS style sheet.
  • A selector defines the element or attributes to which the rules, or declarations will apply.
  • Declarations contain two important components: The CSS property we want to apply and the value of the property.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rules to a class attribute with a class selector.
  • Do Not Repeat Yourself "DRY" is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained.

Git

  • Git is a verson control software and GitHub is a cloud-based service that hosts our repositories.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: commits all files in the current working branch and adds modifications to the staging area.
  • git commit -m "commit message": commits the files.
  • Always do a git pull to make sure that you have the latest version of the project on your local computer before you start to work.
  • git push: pushes the changes we have locally to our remote GitHub branch.
  • A branch is a copy or version of the code that can be worked on independently in isolation so development changes do not disrupt other team members who are also working on the code and testing their solutions.

JavaScript

  • JavaScript is a high dynamic programing language that allows users to interact with websites.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script, however, using conditional statement, allows us to interrupt control flow.
  • The const keyword allows us to create variables that can't be reassigned a value. It "locks down" what that variable will be.
  • An array is a single variable that is used to hold a group of data, such as, strings, numbers, and so on.
  • Array index: to work with individual data items inside an array.
  • A 'for' loop uses predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • 2 requirements for a JavaScript function: define the function and call the function.
